# Build a support agent

This tutorial mirrors the canonical example at
[`examples/support_agent/`](https://github.com/kwameasare/loop/tree/main/examples/support_agent).
By the end you will have:

- A `SupportAgent` class with one tool (`lookup_order`).
- A local provider loop that streams responses from OpenAI or
  Anthropic.
- An eval suite that pins the agent's behaviour for CI.

<Steps>
  <Step title="Define the agent">
    Create `agent.py`:

    ```python
    from loop.agents import Agent

    class SupportAgent(Agent):
        name = "support"
        model = "gpt-4o-mini"
        instructions = "Answer order-status questions politely."
        max_iterations = 4
        max_cost_usd = 0.10
    ```
  </Step>
  <Step title="Add a tool">
    ```python
    async def lookup_order(order_id: str) -> dict:
        # In production this calls your OMS; here we stub it.
        return {"status": "in_transit", "estimated_delivery": "2026-05-02"}

    LOOKUP_ORDER_TOOL_SCHEMA = {
        "name": "lookup_order",
        "description": "Look up the status of a customer order by ID.",
        "input_schema": {
            "type": "object",
            "properties": {"order_id": {"type": "string"}},
            "required": ["order_id"],
        },
    }
    ```
  </Step>
  <Step title="Run locally">
    ```bash
    uv run python examples/support_agent/run_local.py \
      "where is order 4172?"
    ```
  </Step>
  <Step title="Deploy">
    ```bash
    loop deploy --agent support
    ```
    The CLI uploads the agent, the control plane schedules a runtime
    pod, and the data-plane URL becomes addressable from the studio.
  </Step>
  <Step title="Add an eval">
    See the [run-evals tutorial](/tutorials/run-evals) for the
    assertion catalog.
  </Step>
</Steps>
